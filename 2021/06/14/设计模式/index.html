设计模式

发表于 2021-06-14 分类于 Java 阅读次数： Valine：
本文字数： 383 阅读时长 ≈ 1 分钟

关于设计模式，个人的理解就是是一个利用面向对象的三大特点实现开发的一个经典模式。它规定了23种设计模式，大致思想主要体现在有抽象类实现逻辑算法，子类、具体对象实现具体操作。采取设计模式可以降低犯错的几率，提高系统维护及更新能力。

基本原则

开闭原则

一句很抽象的话，对软件实现应该关闭修改，放开扩展。也就是说我们修改一个对象的时候，尽量使用扩展的方式对其进行修改而不是直接修改该对象。接下来5个原则都遵从开闭原则。

里氏替换原则

子类可以扩展父类的功能，但是不能修改父类的功能

迪米特法则(最少知道原则)

一个类不能访问类变量中不依赖的类。

依赖倒置原则

面向接口编程，不要面向实现编程，主张逻辑由抽象接口实现，子类实现细节。

接口隔离原则

一个类A如果依赖于另一个接口B，那么必须依赖于B的所有方法，否则考虑将B进行拆分。

单一职责原则

一个类的职责尽量单一，可以使类的粒度更小，易于复用。
